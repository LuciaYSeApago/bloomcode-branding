## 🌸 Core Colors

| Name | Preview | Hex | Usage |
|:----|:----:|:----:|:----|
| Sakura Pink | 🩷 | `#FF8DC7` | Titles, highlights, accents |
| Lavender | 💜 | `#C792EA` | Secondary headings, decorative elements |
| Mint | 🌿 | `#A6E3A1` | Success messages, Git information |
| Cream | 🌼 | `#F9E2AF` | Warnings, important notes |
| Ivory | 🤍 | `#F8F8F2` | Main text |
| Slate | 🌑 | `#2B2B3C` | Backgrounds and dark UI elements |

---

## 🌸 Design Philosophy

BloomCode uses a pastel-inspired palette to create a calm and welcoming learning environment.

Each color has a clear purpose:

🩷 **Sakura Pink**
- Main identity color
- Titles
- Project names
- Decorative accents

💜 **Lavender**
- Secondary information
- Subtitles
- UI decorations

🌿 **Mint**
- Success
- Git status
- Completed tasks
- Positive feedback

🌼 **Cream**
- Warnings
- Tips
- Things worth remembering

🤍 **Ivory**
- Primary readable text

🌑 **Slate**
- Dark backgrounds
- Terminal themes
- Banner backgrounds

---

## 💻 Terminal Colors

If possible, BloomCode terminal applications should follow this hierarchy:

| Element | Color |
|---------|-------|
| Title | 🩷 Sakura Pink |
| Menu | 💜 Lavender |
| Success | 🌿 Mint |
| Warning | 🌼 Cream |
| Error | ❤️ Red |
| Normal text | 🤍 Ivory |

---

## 📝 README Usage

Every README should primarily use:

- 🩷 Sakura Pink
- 💜 Lavender
- 🤍 Ivory

Green and yellow should be reserved for meaning (success or warnings), not decoration.
