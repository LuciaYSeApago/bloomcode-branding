# 🌸 Purpose

Screenshots should showcase the application clearly while maintaining the cozy BloomCode aesthetic.

The goal is to make projects inviting, readable and consistent.

---

# 💻 Terminal

- Use the BloomCode terminal layout.
- Use the Starship prompt.
- Keep the terminal clean.
- Avoid unnecessary commands in screenshots.
- Display only relevant information.

---

# 🖥️ Window

- Dark theme.
- Consistent window size.
- No personal information.
- No desktop clutter.
- Keep margins similar across projects.

---

# 🎨 Colors

Use the BloomCode palette whenever possible.

🩷 Sakura Pink → Titles

💜 Lavender → Menus

🌿 Mint → Success

🌼 Cream → Warnings

🤍 Ivory → Main text

---

# 📷 Screenshot Types

Every project should ideally include:

## Main Menu

The first screen users see.

---

## Main Functionality

The application's core feature.

---

## Optional

Interesting interactions or additional features.

---

# 📂 Storage

Store every image inside:

screenshots/

Example:

screenshots/
├── main-menu.png
├── gameplay.png
└── settings.png

---

# 🌱 Philosophy

A screenshot should answer one question:

"What does this project look like?"

The cleaner the screenshot, the easier it is to understand the application.

---

> 🌸 Consistency makes every project feel connected.
